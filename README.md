# Realtime Meeting Summary Extension

Google Meetの会議中に、リアルタイムで会議の内容を要約して表示するChrome拡張機能です。
会議の進行に合わせて、議事録が徐々に更新されていくような体験を提供します。

## 機能

- **リアルタイム要約**: Google Meetの会議内容をリアルタイムで要約します。
- **別タブ表示**: 要約を別のタブで大きく表示し、会議の邪魔になりません。
- **手動修正**:
    - 要約テキストを直接編集。
    - プロンプトによる指示で修正。
- **API設定**:
    - Gemini API または ChatGPT API を利用可能。
    - ChatGPT API利用時はモデル選択によるパラメータ調整に対応（予定）。
    - API使用料の概算表示（予定）。

## 検討中の機能

- **発言取得方法**:
    - 音声認識（APIコスト等の検討が必要）
    - 字幕取得（字幕表示の強制等の検討が必要）
- **修正インターフェース**:
    - リアルタイム更新と競合しない編集UI
    - 割り込みプロンプトのタイミング制御

## インストール方法

1. このリポジトリをクローンまたはダウンロードします。
2. Chromeブラウザで `chrome://extensions/` を開きます。
3. 右上の「デベロッパーモード」をオンにします。
4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、ダウンロードしたフォルダを選択します。

## 実装の詳細

### Phase 1: 基本構成の作成
拡張機能の土台となる部分を構築しました。
- **Manifest V3対応**: 最新のChrome拡張機能仕様に準拠した `manifest.json` を作成。
- **ポップアップUI**: 拡張機能アイコンクリック時に表示されるメニュー画面 (`popup.html`) を作成。
- **設定画面**: APIキーなどを設定するためのオプションページ (`options.html`) の枠組みを作成。

### Phase 2: Google Meet 統合
Google Meetの画面上に独自のUIを表示するための仕組みを実装しました。
- **Content Scriptの注入**: `meet.google.com` ドメインでのみ動作するスクリプト (`content_script.js`) を設定。
- **UIインジェクション**: 会議画面のDOMに、要約を表示するためのフローティングパネルを動的に生成・追加。
- **表示切り替え**: 邪魔にならないよう、パネルの最小化/展開ボタンを実装。

### Phase 3: 字幕取得システム
Google MeetのDOM構造を解析し、以下の手法で字幕を取得しています。

1.  **MutationObserverの利用**:
    - `document.body` を監視し、字幕コンテナ（`aria-label="字幕"` または `aria-label="Captions"`）が出現するのを検知します。
    - 字幕コンテナ内のDOM変化を監視し、新しいテキストノードが追加されたタイミングで処理を行います。

2.  **重複とノイズの排除**:
    - **ID管理**: 各字幕ノードに一意のIDを割り振り、`content_script.js` 側でIDに基づいて行の更新・追加を判断します。これにより、Meetが既存の字幕を修正した際にも、行が増えずに正しく更新されます。
    - **デバウンス処理**: 字幕の確定を待つため、500msの遅延（デバウンス）を設けています。
    - **UIノイズ除去**: 「カメラはオン」「マイクはオン」などのシステムメッセージを除外しています。

3.  **話者の特定**:
    - 字幕テキストの近くにある `img` タグ（アバターアイコン）の `alt` 属性から話者名を取得します。
    - 取得できない場合は `Unknown` と表示されます（「あなた」の特定は今後の課題）。

4.  **UX改善**:
    - 字幕がOFFの場合、パネル内に「字幕を開いてください (Shift+C)」という警告を表示し、ユーザーに操作を促します。

### Phase 4: API連携とリアルタイム要約 (完了)
取得した字幕データをOpenAI API (ChatGPT) に送信し、リアルタイムで議事録・要約を生成して表示する機能を実装しました。

1.  **API連携**:
    -   **プロバイダ**: OpenAI API (モデル: `gpt-4o-mini` を採用)
    -   **認証**: ユーザーが設定画面でAPIキーを入力・保存する仕組み。
    -   **通信**: `background.js` を経由してAPIリクエストを行う（セキュリティとCORS対策）。

2.  **要約ロジック (Update Strategy)**:
    -   **差分更新方式**: 「現在の要約」＋「直近の字幕履歴」＋「新しい字幕」をセットでAPIに送信。
    -   **プロンプト**: 「過去の決定事項を維持しつつ、新しい会話を統合して議事録を更新せよ」という指示により、会議が進むにつれて議事録が成長していく体験を実現。
    -   **スライディングウィンドウ**: 文脈を維持するため、前回の字幕（デフォルト5件）を含めて送信。

3.  **カスタマイズとUX**:
    -   **設定画面**:
        -   APIキー設定、モデル選択。
        -   **生成インターバル**: 要約を更新する間隔（秒）を指定可能。
        -   **履歴件数**: 文脈として含める過去の字幕数を指定可能。
        -   **システムプロンプト**: AIへの指示を自由に編集可能。
    -   **メインパネル**:
        -   **ローディング表示**: 生成中にスピナーを表示。
        -   **デバッグハイライト**: AIに送信済みの字幕を青色背景で可視化。
    -   **安全なリクエスト制御**: インターバル経過かつ前のリクエスト完了時のみ次を実行し、多重リクエストを防止。

### Phase 5: 要約の別タブ表示 (完了)

## 今後の予定

### Phase 8: 字幕自動ON機能 (完了)
会議参加時に、ユーザーが手動で字幕をONにする手間を省く機能を実装しました。
- **自動ONロジック**: 会議参加（DOM検知）と同時に、字幕ボタンの状態を確認し、OFFなら自動的にクリックしてONにします。
- **設定オプション**: 設定画面でこの機能のON/OFFを切り替え可能です（デフォルトON）。
